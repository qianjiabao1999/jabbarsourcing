## 3. 技术与设计约定

- **栅格**:内容层只允许 `container-wide`(1140px)与 `container-narrow`(720px),margin:0 auto;全宽背景带除外。
- **品牌色**:主色深青 `--brand-teal`、点缀橙(hero CTA)、WhatsApp 绿 #25b35d;渐变背景必须设纯色 background-color 兜底;禁止新造第二套颜色变量。
- **规范**:主按钮 ≥48px、字号 ≥16px、对比度 ≥4.5:1;正文/FAQ ≥16px;外链一律 `target="_blank" rel="noopener"`;动画尊重 prefers-reduced-motion 且首屏豁免;/ar/ 改动后必须 RTL 自检。
- **FAQPage JSON-LD 与可见文案必须逐字一致**(Google 判罚点)。
- **极简界面原则(站主拍板)**:全站无任何浮动/常驻叠加控件——浮动联系钮、底部转化条、返回顶部、国旗滚动带均为**有意移除**,联系入口 = 吸顶导航 Free Quote + 页脚。QA 反浮动断言是该约定的守卫,勿当死代码清理;验收时勿把这些组件的缺失当回归修复。Turnstile 失败态在表单内渲染的 WhatsApp 兜底链接不属浮动控件,保留。
- **AI 采购助理**:浮钮 class `jabbar-ai-toggle` + 面板,后端 Cloudflare Worker;两条硬规则——回复语言跟随买家最后一条消息;只谈采购话题,跑题一句话拒绝拉回。

## 4. 部署铁律(本仓库与其他项目最大的差异)

1. **push 到 main = 直接上线生产**:deploy.yml 会跑全量 QA(npm test + Playwright 多套 qa:*)后发布 Pages。main 上只允许出现你愿意让全世界访问的状态。
2. **发布走白名单**:deploy.yml 用 rsync include 白名单逐文件列举发布物,并断言 HTML 总数与每语言文件数。**新增任何要上线的页面/资源,必须同步更新 deploy.yml 白名单与计数断言**,否则要么静默不上线、要么 CI 红。`.md` 文件被明确禁止进发布产物(交接文档因此安全)。
3. **styles.min.css 是生成物**:改样式后必须 `npm run build:css` 并提交产物,CI 会校验无 diff。
4. 每次改动后本地先跑 `npm test`;动过询盘/consent/计算器的,跑对应 `npm run qa:*` 套件。

## 5. 红线

1. 本仓库工作只动本仓库;禁止触碰 jabbar-erp、haoduobao 等其他项目的仓库/数据/配置。
2. 禁止改动域名与 DNS 配置;域名三件套(自动续费 + 转移锁 + 本人账户)不容破坏。
3. 禁止提交任何凭据/令牌/内部任务书进本公开仓库;凭据不跨机器拷贝,每台机器各自登录。
4. 未经站主确认禁止删除 QA 断言(尤其防 ja 回归与反浮动断言)。

## 6. 交接纪律(跨两台电脑工作)

只有"已推送的提交"才能跨机,会话内记忆与本机工作区都不算数。

### 6.1 会话收尾义务

1. 覆盖式重写 `HANDOFF.md`(模板见 6.3;只写可公开内容,内部细节用任务书编号指代)。
2. `git status` 清点现场:**完成且过 QA 的改动才允许进 main**(进 main 即上线);未完成的现场推 `wip-*` 分支跨机(`git push origin HEAD:wip-<主题>`),wip 分支不触发部署。
3. 提交信息用中文,小步提交。
4. 推送后确认:进 main 的,盯 Actions 全绿;进 wip 的,报告分支名并写进 HANDOFF.md。

### 6.2 新会话开工流程

1. `git pull`;若 HANDOFF.md 提到 wip 分支,检出或合并现场。
2. 完整阅读本文件 + `HANDOFF.md`,复述现状与计划,确认后动手。
3. 复述与实际不符时以 `git log`、代码实况为准,当场修正 HANDOFF.md。

### 6.3 HANDOFF.md 模板

```markdown

## 1. 当前目标

```

## 8. 历史教训(每条都是真实发生过的返工)

- 渐变背景不设纯色兜底 → 白字白底。
- 新增语言版本从旧模板复制 → 丢失全部历史优化;必须对照里程碑清单逐项同步。
- 手写栅格类名拼错(`contain` vs `container-wide`)→ 改动后必跑栅格自检。
- 第一版入场动画过慢 → "白屏";动画渐进增强、首屏豁免。
